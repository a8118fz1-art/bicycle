V16.35_3STEP_RPM_STABILITY

Scope:
- app.js was not modified.
- Existing FREE RIDE / ERG / TEST / 3-STEP page flow was preserved.
- Changes are limited to ui.js result data handling and display protection.

Fixes:
1. 3-STEP RPM recording now ignores invalid/no-input RPM values below 20 rpm.
2. Step max RPM is calculated only from actual valid RPM samples collected during each 10-second step.
3. Each step stores max RPM, average RPM, and sample count internally for traceability.
4. Result calculation still uses max RPM for Result 1 / 2 / 3, matching the existing result method.
5. Regression result now displays the full formula: rpm = a × kp + b.
6. Calculation status now shows R² for quick data quality review.
7. If any step has no valid RPM sample, the result page remains in NO DATA / Waiting for valid RPM data status and does not generate a chart or power result.

Important:
- No changes were made to app.js.
- No changes were made to FTMS command logic.
- No changes were made to the 3-STEP timing, step transition, or page transition procedure.
